LLMs freeze their knowledge at training time. RAG is how you give a model access to your data — documents, databases, real-time information — without retraining or fine-tuning. It's the architecture behind almost every production AI application.

5.1 The Core RAG Pattern

RAG is an open-book exam for your LLM. Instead of asking it to recall facts from training, you retrieve the relevant "book pages" and put them in the context window alongside the question. The model reads what you gave it and answers — it doesn't need to have memorized it.

# The RAG loop in 4 steps:
#
# 1. INGEST: Chunk your documents → embed each chunk → store in vector DB
# 2. RETRIEVE: Embed the user query → find top-K similar chunks
# 3. AUGMENT: Inject retrieved chunks into the prompt as context
# 4. GENERATE: Model answers using both its training AND your documents

def rag_query(user_question: str) -> str:
    # Step 2: retrieve
    query_embedding = embed(user_question)
    relevant_chunks = vector_db.search(query_embedding, top_k=5)

    # Step 3: augment
    context = "\n\n---\n\n".join(chunk.text for chunk in relevant_chunks)
    prompt = f"""Answer the question using only the context below.
If the answer isn't in the context, say "I don't have that information."

Context:
{context}

Question: {user_question}"""

    # Step 4: generate
    return call(SYSTEM, prompt)
The phrase "using only the context below" is important. Without it, the model will mix retrieved information with hallucinated training knowledge. Grounding the model to your context improves both accuracy and auditability.
5.2 Chunking Strategy: The Hidden Quality Driver

How you split your documents into chunks determines retrieval quality more than any other single factor. Chunks too large include irrelevant text. Chunks too small lose context. The right size depends on your document type and query patterns.

StrategyBest forChunk size
Fixed-size sliding windowGeneric documents, quick setup512–1024 tokens, 10–20% overlap
Sentence-basedProse, articles, legal docs3–5 sentences
Semantic splittingTechnical docs with distinct modulesOne concept per chunk
Recursive character splitCode, structured markdownSplit on newlines → sentences → chars
 
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=800,        # ~200 words, fits well in context
    chunk_overlap=100,     # preserve continuity at boundaries
    separators=["\n\n", "\n", ". ", " ", ""]  # try double newline first
)

chunks = splitter.split_text(document_text)
print(f"Split {len(document_text)} chars into {len(chunks)} chunks")
Always include metadata with each chunk: source document, page number, module title, date. When the model cites its answer, you need to trace it back to the source. Without metadata, your RAG system is a black box.

def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# --- Index: embed all chunks at ingest time ---
class SimpleVectorStore:
    def __init__(self):
        self.chunks = []      # list of {"text": ..., "metadata": ..., "embedding": ...}

    def add(self, text: str, metadata: dict):
        self.chunks.append({
            "text": text,
            "metadata": metadata,
            "embedding": embed(text)
        })

    def search(self, query: str, top_k: int = 5) -> list[dict]:
        q_emb = embed(query)
        scored = [
            (cosine_similarity(q_emb, c["embedding"]), c)
            for c in self.chunks
        ]
        scored.sort(reverse=True, key=lambda x: x[0])
        return [c for _, c in scored[:top_k]]

5.4 Verify: Working RAG Pipeline

Test your RAG pipeline on a document you know well, so you can verify whether the retrieved chunks are actually the right ones — not just whether the final answer sounds plausible.

The fastest way to catch retrieval bugs: print the retrieved chunks before the model sees them. If you're seeing chunks that obviously don't answer the question, your chunking or embedding strategy needs work — not the prompt.
