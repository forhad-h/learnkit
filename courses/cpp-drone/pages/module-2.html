Do this before learning C++. Running a simulated drone on the machine before understanding everything creates a concrete target — every concept learned afterward feels relevant instead of abstract.
2.1 Install ROS2 Jazzy

ROS2 Jazzy is the LTS release for Ubuntu 24.04. Run each command separately.

# Step 1 — Install prerequisites
sudo apt install software-properties-common curl -y
# Step 2 — Add the GPG key
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key \
  -o /usr/share/keyrings/ros-archive-keyring.gpg
# Step 3 — Add the ROS2 repository
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null
# Step 4 — Install
sudo apt update && sudo apt install ros-jazzy-desktop -y
# Step 5 — Source on every terminal session
echo "source /opt/ros/jazzy/setup.bash" >> ~/.bashrc
source ~/.bashrc
# Step 6 — Verify
ros2 --version

Test that ROS2 works

# Terminal 1
ros2 run demo_nodes_cpp talker

# Terminal 2
ros2 run demo_nodes_cpp listener
# Expected: [INFO] I heard: Hello World: 1, 2, 3...
If the listener prints "Hello World" — ROS2 Jazzy is fully installed.
2.2 Install ArduPilot SITL

SITL (Software In The Loop) runs a complete virtual drone on the machine — same code as real hardware, without physical motors.

git clone https://github.com/ArduPilot/ardupilot.git
cd ardupilot
git submodule update --init --recursive
Tools/environment_install/install-prereqs-ubuntu.sh -y
. ~/.profile
# Build takes 5–15 minutes
./waf configure --board sitl
./waf copter
# Run the simulation
sim_vehicle.py -v ArduCopter --console --map
# Arm and take off — type in the MAVProxy terminal:
mode guided
arm throttle
takeoff 20
✅ Milestone: A virtual drone is flying on screen. Everything in this course runs here first.
2.3 Set Up a Workspace

All ROS2 code lives in a workspace. Create this once — every project in this course goes inside it.

mkdir -p ~/drone_ws/src
echo "cd ~/drone_ws && source install/setup.bash 2>/dev/null; cd ~" >> ~/.bashrc
~/drone_ws/
  src/          ← all packages go here
  build/        ← generated, do not edit
  install/      ← generated, do not edit
  log/          ← generated, do not edit
The src/ folder is the only one you create manually. The others are generated by colcon build.
