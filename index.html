Understanding the Drone Software Stack
Read first

A drone is not one program — it is a collection of programs working together in real time. Understanding the layer structure before writing code prevents confusion about where each technology fits.

Layer 4  ·  AI / Computer Vision    Object detection, tracking, ML models
Layer 3  ·  Autonomy & Planning     ROS2, waypoints, state machines   ← This tutorial starts here
Layer 2  ·  Flight Control          PX4 / ArduPilot, MAVLink, PID loops
Layer 1  ·  Hardware / Embedded     RTOS, bare metal C++, sensor drivers
Why C++? Every layer except Layer 4 is dominated by C++. ROS2, PX4, and ArduPilot are all written in C++. Python handles the AI layer and scripting; flight-critical real-time code is C++.
Why ROS2? ROS2 is the industry-standard communication framework connecting all drone components — camera, GPS, lidar, flight controller — without hard-coding them together.

Learning order in this tutorial

Environment setup → C fundamentals refresh → C++ basics → ROS2 concepts
→ First node → 4 portfolio projects → Read real code → Contribute

C Fundamentals Refresher
Skip if confident
Read this if pointers, memory, or structs feel uncertain. These are the foundation that makes C++ make sense. Skip to Section 3 if all five concepts are solid.
2.5.1 Memory: Stack vs Heap
#include <stdio.h>
#include <stdlib.h>

void demonstrate_memory() {
    // STACK — automatic, fast, limited (~1–8MB)
    int altitude = 100;
    float battery[4] = {87.2, 86.1, 85.0, 83.9};
    // freed automatically when function returns

    // HEAP — manual, slower, much larger
    float* sensor_readings = malloc(10 * sizeof(float));
    sensor_readings[0] = 23.8;
    free(sensor_readings);   // MUST free — forget = memory leak
    sensor_readings = NULL;  // good practice
}
Never return a pointer to a stack variable. The stack is destroyed when the function returns. 
float* bad()  { float x = 100.0; return &x; }  // WRONG — dangling pointer
float* good() { float* x = malloc(sizeof(float)); *x = 100.0; return x; } // OK

Why this matters: C++ smart pointers (Section 3.5) automate the free() step — but only make sense if the underlying concept is understood.

2.5.2 Pointers
float altitude = 100.0;
float* ptr = &altitude;  // & = "address of"

printf("%.1f\n", *ptr);  // 100.0  (* = dereference)
*ptr = 200.0;
printf("%.1f\n", altitude);  // 200.0 — original changed

// NULL — always check before dereferencing
float* empty = NULL;
if (empty != NULL) { printf("%.1f\n", *empty); }
// Dereferencing NULL = segfault = most common drone code crash
// Pointer arithmetic — used in sensor data buffers
float readings[5] = {10.0, 20.0, 30.0, 40.0, 50.0};
float* p = readings;
printf("%.1f\n", *(p+1));  // 20.0
printf("%.1f\n", readings[1]);  // same thing
2.5.3 Structs
typedef struct {
    double latitude;
    double longitude;
    float  altitude;
} GPSPosition;

// Pass by pointer — avoids copying large structs
void update_altitude(GPSPosition* pos, float new_alt) {
    pos->altitude = new_alt;  // -> for pointer to struct
}

GPSPosition home = {23.8103, 90.4125, 10.0};
update_altitude(&home, 25.0);
printf("%.1f\n", home.altitude);  // 25.0

Why this matters: Every ROS2 message works exactly like this struct. sensor_msgs::msg::NavSatFix has latitude, longitude, altitude fields — identical concept.

2.5.4 Functions and Scope
// Pass by value — function gets a COPY, original unchanged
void reset_by_value(float altitude) { altitude = 0.0; }

// Pass by pointer — can modify original
void reset_by_pointer(float* altitude) { *altitude = 0.0; }

float alt = 100.0;
reset_by_value(alt);    printf("%.1f\n", alt);  // 100.0 — unchanged
reset_by_pointer(&alt); printf("%.1f\n", alt);  // 0.0 — changed
// Output parameter — returning multiple values
void get_gps_error(float* h_err, float* v_err) {
    *h_err = 2.5;
    *v_err = 4.1;
}

float h, v;
get_gps_error(&h, &v);  // h=2.5, v=4.1
2.5.5 Header Files and Compilation Units
// gps.h — declares what exists
#ifndef GPS_H
#define GPS_H
typedef struct { double lat; double lon; float alt; } GPSPos;
float distance(GPSPos a, GPSPos b);  // declaration only
#endif
// gps.c — defines how it works
#include "gps.h"
#include <math.h>
float distance(GPSPos a, GPSPos b) {
    double dlat = a.lat - b.lat;
    double dlon = a.lon - b.lon;
    return (float)sqrt(dlat*dlat + dlon*dlon);
}
# Compile multiple files
gcc main.c gps.c -o drone_nav -lm

Why this matters: CMakeLists.txt in ROS2 does exactly this — specifies which .cpp files to compile and which libraries to link. Understanding this makes CMake errors readable.

C++ Fundamentals for Drone Development
Core skill
3.1 Why C++ Feels Hard + Compilation Basics

C++ has three things higher-level languages hide: manual memory management, compilation, and explicit types everywhere. These are features — they make C++ fast enough for real-time drone control.

cat > hello.cpp << 'EOF'
#include <iostream>
int main() {
    std::cout << "Drone software starts here" << std::endl;
    return 0;
}
EOF
g++ hello.cpp -o hello
./hello

Unlike Python, C++ must be compiled before running. Bugs caught at compile time cannot crash a drone in flight.

3.2 Classes

A class is a struct with functions attached and access control. Every ROS2 node, every sensor driver, every controller is a class.

#include <cmath>
class Point {
public:
    Point(float x, float y) : x_(x), y_(y) {}  // constructor

    float distance_to_origin() const {
        return std::sqrt(x_*x_ + y_*y_);
    }
    float x() const { return x_; }

private:
    float x_, y_;  // only accessible inside the class
};

Point p(3.0f, 4.0f);
p.distance_to_origin();  // 5.0

Practice: Write a Waypoint class with latitude, longitude, altitude, and a method that calculates distance to another waypoint.

3.3 References

References are cleaner syntax for the same idea as C pointers — passing by address without the * and & noise.

// C style
void set_c(float* altitude, float value) { *altitude = value; }
set_c(&alt, 100.0f);

// C++ style — cleaner, same effect
void set_cpp(float& altitude, float value) { altitude = value; }
set_cpp(alt, 100.0f);

Rule: Prefer references over pointers. Use pointers only when the value can be absent (nullptr) or when transferring ownership.

3.4 Smart Pointers

Smart pointers automatically call delete when they go out of scope. Used in every single ROS2 node.

#include <memory>

// unique_ptr — one owner, freed when out of scope
auto p = std::make_unique<float>(100.0f);
// no delete needed

// shared_ptr — multiple owners, freed when last owner dies
// THIS IS EVERYWHERE IN ROS2
auto p = std::make_shared<float>(100.0f);
auto also_p = p;  // both point to same memory
This line appears in every ROS2 node: 
rclcpp::Publisher<sensor_msgs::msg::NavSatFix>::SharedPtr publisher_;
//                                                          ↑ this is just shared_ptr
3.5 STL Containers

Do not use raw C arrays in modern C++.

#include <vector>
#include <map>
#include <string>
#include <array>

std::vector<float> altitudes = {10.0, 20.0, 30.0};
altitudes.push_back(40.0);
for (float alt : altitudes) { /* range loop */ }

std::array<double, 3> waypoint = {23.8103, 90.4125, 50.0};
// [0]=lat  [1]=lon  [2]=altitude

std::map<std::string, float> sensor;
sensor["altitude"] = 100.5f;

std::string id = "UAV_001";
std::string full = id + "_SURVEY";  // "UAV_001_SURVEY"
3.6 Lambdas

JavaScript arrow function = Python lambda = C++ lambda. Every ROS2 callback uses this pattern.

auto add = [](int a, int b) { return a + b; };
add(3, 4);  // 7

// Capture outer variable
float threshold = 5.0f;
auto is_close = [threshold](float dist) { return dist < threshold; };

// [this] capture — used in every ROS2 node
timer_ = create_wall_timer(100ms, [this]() { publish_data(); });

Understanding ROS2
Core framework

ROS2 is a communication middleware. It lets dozens of separate drone programs exchange data in real time without being hard-coded to each other.

4.1 Core Concepts: Nodes, Topics, Messages, Services
WITHOUT ROS2                    WITH ROS2
────────────────────────────    ────────────────────────────────
camera → flight (custom code)   camera_node  publishes → /camera/image
gps → autopilot (custom code)   vision_node  subscribes ← /camera/image
                                             publishes  → /target/position
Everything tightly coupled.     autopilot_node subscribes ← /target/position
Change one = break others.                     publishes  → /cmd_vel
                                Loosely coupled. Add/remove nodes freely.

Node

A single executable process with one responsibility. Nodes run independently.

ros2 node list
ros2 node info /node_name

Topic

A named data stream. Any node can publish or subscribe. Publishers and subscribers don't know about each other — only the topic name.

ros2 topic list
ros2 topic echo /gps/fix
ros2 topic hz /camera/image_raw
ros2 topic pub /test std_msgs/msg/String "data: hello"

First ROS2 Node in C++
Memorize this

Every ROS2 C++ node follows this exact structure. Learn it once — it applies everywhere.

5.1 Universal Node Pattern
#include "rclcpp/rclcpp.hpp"
#include "some_msgs/msg/some_type.hpp"

// Step 1: Inherit from rclcpp::Node
class MyNode : public rclcpp::Node {
public:
    // Step 2: Constructor — set up publishers, subscribers, timers
    MyNode() : Node("my_node_name") {
        publisher_ = create_publisher<MsgType>("/topic", 10);

        subscriber_ = create_subscription<MsgType>(
            "/other_topic", 10,
            [this](const MsgType::SharedPtr msg) { on_message(msg); }
        );

        timer_ = create_wall_timer(
            std::chrono::milliseconds(100),
            [this]() { on_timer(); }
        );
        RCLCPP_INFO(get_logger(), "Node started");
    }

private:
    // Step 3: Callbacks
    void on_message(const MsgType::SharedPtr msg) { /* handle */ }
    void on_timer() { publisher_->publish(MsgType()); }

    // Step 4: Members — always SharedPtr
    rclcpp::Publisher<MsgType>::SharedPtr publisher_;
    rclcpp::Subscription<MsgType>::SharedPtr subscriber_;
    rclcpp::TimerBase::SharedPtr timer_;
};

// Step 5: main — always identical
int main(int argc, char* argv[]) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<MyNode>());
    rclcpp::shutdown();
    return 0;
}

class TelemetryNode : public rclcpp::Node {
public:
    TelemetryNode() : Node("telemetry") {
        publisher_ = create_publisher<sensor_msgs::msg::NavSatFix>("/gps/fix", 10);
        timer_ = create_wall_timer(100ms, [this]() { publish_gps(); });
    }
private:
    void publish_gps() {
        auto msg = sensor_msgs::msg::NavSatFix();
        msg.latitude  = 23.8103 + (counter_ * 0.0001);
        msg.longitude = 90.4125 + (counter_ * 0.0001);
        msg.altitude  = 100.0;
        publisher_->publish(msg);
        counter_++;
    }
    rclcpp::Publisher<sensor_msgs::msg::NavSatFix>::SharedPtr publisher_;
    rclcpp::TimerBase::SharedPtr timer_;
    int counter_ = 0;
};
Project 2 — Obstacle Detection Subscriber   1 week

What it does: Reads lidar data, logs warning when obstacle is within 5m.
Concepts: Subscribers, callbacks, processing sensor arrays, invalid reading filtering.

void process_scan(const sensor_msgs::msg::LaserScan::SharedPtr msg) {
    float closest = std::numeric_limits<float>::max();
    for (float r : msg->ranges) {
        if (std::isfinite(r) && r >= msg->range_min && r <= msg->range_max)
            closest = std::min(closest, r);
    }
    if (closest < 5.0f)
        RCLCPP_WARN(get_logger(), "OBSTACLE at %.2fm!", closest);
}

Project 4 — PID Altitude Controller   1 week

What it does: Maintains target altitude using a PID controller.
Concepts: Control theory, real-time constraints, floating point precision, anti-windup.

class PIDController {
public:
    PIDController(float kp, float ki, float kd, float min, float max)
        : kp_(kp), ki_(ki), kd_(kd), min_(min), max_(max) {}

    float compute(float setpoint, float measurement, float dt) {
        float error = setpoint - measurement;
        integral_ = std::clamp(integral_ + error * dt, -10.0f, 10.0f);
        float d = (dt > 0) ? (error - prev_error_) / dt : 0;
        prev_error_ = error;
        return std::clamp(kp_*error + ki_*integral_ + kd_*d, min_, max_);
    }
private:
    float kp_, ki_, kd_, min_, max_;
    float integral_ = 0, prev_error_ = 0;
};

PID Tuning Guide

1. Start: Kp=1.0, Ki=0.0, Kd=0.0
2. Increase Kp until response is fast but starts to oscillate
3. Add Kd=0.1 to dampen oscillation
4. Add Ki=0.001 only if there is steady-state error
