3.2 Classes

A class is a struct with functions attached and access control. Every ROS2 node, every sensor driver, every controller is a class.

#include <cmath>
class Point {
public:
    Point(float x, float y) : x_(x), y_(y) {}
    float distance_to_origin() const { return std::sqrt(x_*x_ + y_*y_); }
    float x() const { return x_; }
    float y() const { return y_; }
private:
    float x_, y_;
};

int main() {
    Point p(3.0f, 4.0f);
    p.distance_to_origin();  // 5.0
    return 0;
}

3.3 References

References are cleaner syntax for C pointers — passing by address without the * and & noise.

// C style
void set_c(float* altitude, float value) { *altitude = value; }

// C++ style — cleaner, same effect
void set_cpp(float& altitude, float value) { altitude = value; }

int main() {
    float alt = 0.0f;
    set_c(&alt, 100.0f);
    set_cpp(alt, 100.0f);
    return 0;
}

Rule: Prefer references over pointers. Use pointers only when the value can be absent (nullptr) or when transferring ownership.

3.4 Smart Pointers

Smart pointers automatically call delete when they go out of scope. Used in every single ROS2 node.

#include <memory>

int main() {
    // unique_ptr — one owner, freed when out of scope
    auto p = std::make_unique<float>(100.0f);

    // shared_ptr — multiple owners, freed when last owner dies
    // THIS IS EVERYWHERE IN ROS2
    auto sp = std::make_shared<float>(100.0f);
    auto also_p = sp;  // both point to same memory
    return 0;
}

3.6 Lambdas

JavaScript arrow function = Python lambda = C++ lambda. Every ROS2 callback uses this pattern.

int main() {
    auto add = [](int a, int b) { return a + b; };
    add(3, 4);  // 7

    // Capture outer variable
    float threshold = 5.0f;
    auto is_close = [threshold](float dist) { return dist < threshold; };
    return 0;
}

// [this] capture — used in every ROS2 node (inside a class constructor)
// timer_ = create_wall_timer(100ms, [this]() { publish_data(); });
Lambda syntax: [capture](params) { body }
[] — nothing · [=] — all by value · [&] — all by ref · [this] — class instance (most common in ROS2)
