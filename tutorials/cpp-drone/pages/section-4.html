ROS2 is a communication middleware. It lets dozens of separate drone programs exchange data in real time without being hard-coded to each other.

4.1 Nodes, Topics, Messages, Services
WITHOUT ROS2                    WITH ROS2
────────────────────────────    ────────────────────────────────
camera → flight (custom code)   camera_node  publishes → /camera/image
gps → autopilot (custom code)   vision_node  subscribes ← /camera/image
Everything tightly coupled.                  publishes  → /target/position
Change one = break others.      autopilot_node subscribes ← /target/position
                                Loosely coupled. Add/remove nodes freely.

Topics

ros2 topic list
ros2 topic echo /gps/fix
ros2 topic hz /camera/image_raw
ros2 topic pub /test std_msgs/msg/String "data: hello"

Services (request/response)

ros2 service list
ros2 service call /arm_motors std_srvs/srv/Trigger

Use topics for continuous data. Use services for one-time operations (arm, mode change).

4.2 Daily CLI Commands
# Nodes
ros2 node list                     # what's running
ros2 node info /name               # inputs, outputs

# Topics
ros2 topic list                    # all active topics
ros2 topic echo /topic             # see live data
ros2 topic hz /topic               # publish rate

# Build
colcon build                       # build everything
colcon build --packages-select pkg # build one package
source install/setup.bash          # ALWAYS after build

# Run
ros2 run pkg_name executable
ros2 launch pkg_name file.py

# Debug
rqt_graph                          # visual node/topic map
ros2 bag record -a                 # record everything
