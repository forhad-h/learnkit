Defense companies and serious robotics employers know ArduPilot, PX4, and ROS2 by name. A contribution — even a documentation fix — signals that the candidate has read real production code and engaged with the community.

# Contribution ladder — start at Level 1, do not skip
Level 1: Fix a typo or grammar error in documentation
Level 2: Improve an unclear or outdated code comment
Level 3: Fix a small bug linked to an existing issue
Level 4: Add a unit test for an untested function
Level 5: Implement a feature from the issue tracker
